# №12: Сценарии использования приложения (Use Cases)

---

**Назначение раздела:** Сценарии использования (Use Cases) описывают взаимодействие пользователей и внешних систем с приложением для достижения конкретных целей. В отличие от бизнес-сценариев (№7), которые фокусируются на бизнес-ценности и критических потоках, use cases детализируют шаги взаимодействия на уровне пользовательского интерфейса и API. Use cases служат основой для разработки функциональных тестов, UX-дизайна и документации для разработчиков.

---

## 1. Методология описания Use Cases

Каждый use case описывается по следующей структуре:

| Параметр | Описание |
|:---|:---|
| **ID** | Уникальный идентификатор (UC-XXX) |
| **Название** | Краткое название сценария |
| **Описание** | Что делает пользователь и какую цель достигает |
| **Первичный актор** | Кто инициирует сценарий (пользователь, партнёр, система) |
| **Вторичные акторы** | Кто участвует в сценарии (другие пользователи, внешние системы) |
| **Предусловия** | Что должно быть выполнено до начала сценария |
| **Основной поток** | Последовательность шагов |
| **Альтернативные потоки** | Варианты отклонения от основного сценария |
| **Постусловия** | Состояние системы после завершения |
| **Бизнес-цель** | Связанная бизнес-цель |
| **Связанные ФТ** | Функциональные требования |
| **Связанные НФТ** | Нефункциональные требования |

## 2. Сценарии для конечных пользователей

### UC-01: Регистрация нового пользователя

| Параметр | Значение |
|:---|:---|
| **ID** | UC-01 |
| **Название** | Регистрация нового пользователя |
| **Описание** | Пользователь создаёт учётную запись в приложении для получения доступа к функциям трекинга, социальным возможностям и персонализированным рекомендациям. |
| **Первичный актор** | Пользователь |
| **Вторичные акторы** | Система аутентификации (Keycloak/Auth0) |
| **Предусловия** | Приложение установлено на устройстве; интернет-соединение доступно |
| **Основной поток** | 1. Пользователь открывает приложение и нажимает "Зарегистрироваться".<br>2. Система отображает форму регистрации (email, пароль, имя, согласие с условиями).<br>3. Пользователь заполняет форму и нажимает "Создать аккаунт".<br>4. Система валидирует данные и отправляет подтверждение на email.<br>5. Пользователь переходит по ссылке в email для подтверждения.<br>6. Система активирует учётную запись и перенаправляет пользователя на экран заполнения профиля.<br>7. Пользователь заполняет профиль (возраст, вес, уровень подготовки, цели).<br>8. Система сохраняет профиль и отображает главный экран приложения. |
| **Альтернативные потоки** | 1. Регистрация через социальные сети (Google, Apple, Facebook).<br>2. Email уже зарегистрирован — система отображает ошибку.<br>3. Пользователь не подтверждает email в течение 24 часов — система отправляет напоминание. |
| **Постусловия** | Пользователь зарегистрирован, профиль создан, доступ к приложению открыт |
| **Бизнес-цель** | Все (БТ-1, БТ-2, БТ-3, БТ-4) |
| **Связанные ФТ** | ФТ-1 (базовая регистрация из ФТ-1 в контексте профиля) |
| **Связанные НФТ** | НФТ-13 (аутентификация), НФТ-31 (юзабилити), НФТ-40 (хранение данных) |

### UC-02: Запись тренировки с подключением датчиков

| Параметр | Значение |
|:---|:---|
| **ID** | UC-02 |
| **Название** | Запись тренировки с подключением датчиков |
| **Описание** | Пользователь подключает внешние устройства (пульсометр, GPS-часы) и записывает тренировку. Система отслеживает показатели в реальном времени и сохраняет их после завершения. |
| **Первичный актор** | Пользователь |
| **Вторичные акторы** | Внешние устройства (BLE/ANT+), Сервис Tracking, Сервис Gamification |
| **Предусловия** | Пользователь авторизован; устройство поддерживает BLE/ANT+; включён Bluetooth |
| **Основной поток** | 1. Пользователь открывает экран тренировки и нажимает "Подключить устройства".<br>2. Система сканирует доступные устройства и отображает их список.<br>3. Пользователь выбирает устройство (пульсометр, GPS-часы).<br>4. Система подключается к устройству и отображает статус "Подключено".<br>5. Пользователь выбирает тип тренировки (бег, велосипед, плавание, йога и т.д.) и нажимает "Начать".<br>6. Система начинает запись: фиксирует время, GPS-трек, пульс, темп, калории (ФТ-1).<br>7. В процессе тренировки система отображает текущие показатели на экране.<br>8. Пользователь завершает тренировку, нажимая "Стоп".<br>9. Система сохраняет все данные, вычисляет итоговые метрики.<br>10. Система обновляет статистику пользователя (ФТ-3), XP и достижения (ФТ-6).<br>11. Система отображает экран с итогами тренировки. |
| **Альтернативные потоки** | 1. Устройство не найдено — система предлагает записать тренировку без датчиков.<br>2. Потеря соединения с датчиком во время тренировки — система продолжает запись с доступными данными.<br>3. Отсутствие интернета — система сохраняет тренировку локально и синхронизирует позже (ФТ-1 офлайн).<br>4. Пользователь ставит тренировку на паузу. |
| **Постусловия** | Тренировка сохранена; статистика обновлена; XP и достижения начислены |
| **Бизнес-цель** | БТ-2 (AI-коучинг) |
| **Связанные ФТ** | ФТ-1 (Запись тренировок), ФТ-2 (Подключение устройств), ФТ-3 (Сравнение), ФТ-6 (Геймификация) |
| **Связанные НФТ** | НФТ-01 (время отклика), НФТ-37 (поддержка протоколов), НФТ-50 (офлайн-режим) |

### UC-03: Получение персонализированного тренировочного плана

| Параметр | Значение |
|:---|:---|
| **ID** | UC-03 |
| **Название** | Получение персонализированного тренировочного плана |
| **Описание** | Пользователь запрашивает автоматически сгенерированный тренировочный план на основе своих целей, уровня подготовки, инвентаря и истории тренировок. |
| **Первичный актор** | Пользователь |
| **Вторичные акторы** | Сервис AI/ML, Сервис Users & Profiles, Сервис Tracking |
| **Предусловия** | Пользователь авторизован; заполнен профиль; записано минимум 3–5 тренировок; добавлен инвентарь |
| **Основной поток** | 1. Пользователь открывает раздел "Планы" в приложении.<br>2. Система отображает текущий план (если есть) и кнопку "Создать новый".<br>3. Пользователь нажимает "Создать новый план".<br>4. Система запрашивает параметры: период (неделя/месяц), цели (улучшение выносливости, скорость, подготовка к марафону и т.д.).<br>5. Пользователь выбирает параметры и нажимает "Сгенерировать".<br>6. Система (AI/ML-сервис) анализирует: цели, уровень подготовки, инвентарь, историю тренировок, данные с датчиков (ФТ-4).<br>7. Система генерирует план с днями тренировок, типами нагрузок, интенсивностью, днями отдыха.<br>8. Система отображает план пользователю с возможностью редактирования.<br>9. Пользователь утверждает план или вносит корректировки.<br>10. Система сохраняет план и добавляет его в календарь пользователя. |
| **Альтернативные потоки** | 1. Недостаточно данных — система предлагает общий план для новичков.<br>2. Пользователь прерывает генерацию — система сохраняет текущий статус.<br>3. Пользователь запрашивает перегенерацию с другими параметрами. |
| **Постусловия** | План создан, сохранён, добавлен в календарь пользователя |
| **Бизнес-цель** | БТ-2 (AI-коучинг) |
| **Связанные ФТ** | ФТ-4 (AI-формирование плана), ФТ-7 (Инвентарь), ФТ-1 (История тренировок) |
| **Связанные НФТ** | НФТ-02 (время генерации плана), НФТ-05 (пропускная способность) |

### UC-04: Сравнение тренировки с прошлыми результатами

| Параметр | Значение |
|:---|:---|
| **ID** | UC-04 |
| **Название** | Сравнение тренировки с прошлыми результатами |
| **Описание** | После завершения тренировки пользователь просматривает сравнение текущих показателей с историческими данными: динамику темпа, пульса, дистанции, VO2max. |
| **Первичный актор** | Пользователь |
| **Вторичные акторы** | Сервис Tracking |
| **Предусловия** | Пользователь авторизован; завершена тренировка (UC-02) |
| **Основной поток** | 1. После завершения тренировки (UC-02) система отображает экран с итогами.<br>2. На экране итогов система показывает кнопку "Сравнить с собой".<br>3. Пользователь нажимает "Сравнить с собой".<br>4. Система загружает исторические данные на аналогичных дистанциях/маршрутах (ФТ-3).<br>5. Система отображает графики: динамика темпа, пульса, VO2max.<br>6. Система показывает персональные рекорды и прогресс.<br>7. Пользователь может выбрать другие метрики или периоды для сравнения. |
| **Альтернативные потоки** | 1. Недостаточно исторических данных — система отображает сообщение "Запишите больше тренировок для сравнения".<br>2. Пользователь может сравнить с регионом (ФТ-19) или профессионалами (ФТ-20). |
| **Постусловия** | Пользователь получил визуализацию прогресса |
| **Бизнес-цель** | БТ-2 (AI-коучинг) |
| **Связанные ФТ** | ФТ-3 (Сравнение с собой), ФТ-1 (История тренировок) |
| **Связанные НФТ** | НФТ-01 (время отклика) |

### UC-05: Создание и участие в социальной группе

| Параметр | Значение |
|:---|:---|
| **ID** | UC-05 |
| **Название** | Создание и участие в социальной группе |
| **Описание** | Пользователь создаёт или присоединяется к социальной группе по интересам (вид спорта, локация, уровень), общается с участниками, публикует посты. |
| **Первичный актор** | Пользователь |
| **Вторичные акторы** | Другие пользователи, Сервис Social, Сервис Notification |
| **Предусловия** | Пользователь авторизован |
| **Основной поток** | 1. Пользователь открывает раздел "Сообщества" или "Группы".<br>2. Система отображает список рекомендованных групп и поиск.<br>3. Пользователь либо ищет группу по интересам, либо нажимает "Создать группу".<br>4. Для создания: пользователь заполняет название, описание, категорию, теги, уровень подготовки, локацию.<br>5. Система создаёт группу и назначает пользователя администратором.<br>6. Для вступления: пользователь находит группу и нажимает "Вступить" (для открытой группы) или отправляет запрос (для закрытой).<br>7. Система добавляет пользователя в группу (или администратор одобряет заявку).<br>8. Пользователь может: публиковать посты, комментировать, общаться в чате (ФТ-5).<br>9. Система отображает ленту активности группы. |
| **Альтернативные потоки** | 1. Закрытая группа — запрос на вступление ожидает одобрения администратора.<br>2. Пользователь может выйти из группы.<br>3. Администратор может назначить модераторов. |
| **Постусловия** | Группа создана или пользователь добавлен в группу |
| **Бизнес-цель** | БТ-1 (Удержание через социум) |
| **Связанные ФТ** | ФТ-5 (Социальные группы), ФТ-15 (Уведомления) |
| **Связанные НФТ** | НФТ-05 (поиск), НФТ-31 (юзабилити) |

### UC-06: Участие в челлендже

| Параметр | Значение |
|:---|:---|
| **ID** | UC-06 |
| **Название** | Участие в челлендже |
| **Описание** | Пользователь присоединяется к индивидуальному или командному челленджу, отслеживает прогресс и получает награды по завершении. |
| **Первичный актор** | Пользователь |
| **Вторичные акторы** | Другие пользователи, Сервис Gamification, Сервис Notification, Сервис Tracking |
| **Предусловия** | Пользователь авторизован; челлендж активен |
| **Основной поток** | 1. Пользователь открывает раздел "Челленджи".<br>2. Система отображает доступные индивидуальные и командные челленджи.<br>3. Пользователь выбирает челлендж и нажимает "Участвовать".<br>4. Для командного челленджа пользователь создаёт команду или присоединяется к существующей.<br>5. Система добавляет пользователя в челлендж и отображает прогресс.<br>6. Пользователь записывает тренировки (UC-02) — система автоматически обновляет прогресс в челлендже.<br>7. Система отображает рейтинг участников в реальном времени.<br>8. Пользователь получает уведомления о прогрессе команды и достижениях других участников.<br>9. По окончании челленджа система подводит итоги и начисляет награды (XP, бейджи, монеты). |
| **Альтернативные потоки** | 1. Пользователь может покинуть челлендж до завершения (награды не начисляются).<br>2. При недостаточной активности пользователь может быть исключён. |
| **Постусловия** | Челлендж завершён; награды начислены |
| **Бизнес-цель** | БТ-1 (Удержание через социум) |
| **Связанные ФТ** | ФТ-6 (Геймификация), ФТ-1 (Запись тренировок), ФТ-15 (Уведомления) |
| **Связанные НФТ** | НФТ-04 (обновление в реальном времени) |

### UC-07: Получение рекомендации по замене инвентаря и покупка

| Параметр | Значение |
|:---|:---|
| **ID** | UC-07 |
| **Название** | Получение рекомендации по замене инвентаря и покупка |
| **Описание** | Система уведомляет пользователя о необходимости замены инвентаря (износ обуви), рекомендует новые модели и позволяет совершить покупку в интернет-магазине. |
| **Первичный актор** | Пользователь |
| **Вторичные акторы** | Система (уведомления), Сервис Commerce, Сервис Tracking, Legacy e-commerce |
| **Предусловия** | Пользователь авторизован; в инвентарь добавлена обувь (ФТ-7); пробег превысил критическое значение |
| **Основной поток** | 1. Система автоматически отслеживает пробег обуви на основе записей тренировок.<br>2. При достижении критического значения (например, 800 км) система отправляет push-уведомление и in-app уведомление.<br>3. Пользователь нажимает на уведомление и открывает раздел "Инвентарь".<br>4. Система отображает сообщение: "Ваши кроссовки изношены (пробег: 800 км)".<br>5. Система показывает рекомендации новых моделей с учётом стиля бега пользователя (ФТ-8).<br>6. Пользователь выбирает модель и нажимает "Купить".<br>7. Система бесшовно перенаправляет пользователя в интернет-магазин (единый логин, единая корзина).<br>8. Пользователь оформляет заказ и оплачивает.<br>9. Система получает подтверждение о покупке и предлагает добавить новый инвентарь в профиль. |
| **Альтернативные потоки** | 1. Пользователь откладывает покупку — система отправляет напоминание через 1–2 недели.<br>2. У пользователя отключены уведомления — рекомендация отображается в разделе "Инвентарь". |
| **Постусловия** | Покупка совершена; новый инвентарь добавлен в профиль |
| **Бизнес-цель** | БТ-3 (Монетизация) |
| **Связанные ФТ** | ФТ-7 (Инвентарь), ФТ-8 (Рекомендации), ФТ-1 (Запись тренировок) |
| **Связанные НФТ** | НФТ-01 (время отклика), НФТ-11 (безопасность платежей) |

### UC-08: Оформление премиум-подписки

| Параметр | Значение |
|:---|:---|
| **ID** | UC-08 |
| **Название** | Оформление премиум-подписки |
| **Описание** | Пользователь оформляет платную подписку для доступа к премиум-функциям (AI-коучинг, расширенная статистика). |
| **Первичный актор** | Пользователь |
| **Вторичные акторы** | Сервис Commerce, Платёжная система (Stripe/Apple Pay/Google Pay) |
| **Предусловия** | Пользователь авторизован; премиум-функции доступны в регионе |
| **Основной поток** | 1. Пользователь нажимает на заблокированную премиум-функцию или открывает раздел "Подписка".<br>2. Система отображает тарифы и список премиум-возможностей.<br>3. Пользователь выбирает тариф и нажимает "Оформить подписку".<br>4. Система перенаправляет на страницу оплаты.<br>5. Пользователь подтверждает платёж.<br>6. Система активирует подписку и открывает доступ к премиум-функциям.<br>7. Система отправляет подтверждение на email пользователя. |
| **Альтернативные потоки** | 1. Платёж отклонён — система отображает ошибку и предлагает повторить.<br>2. Пользователь отменяет подписку — доступ сохраняется до окончания оплаченного периода. |
| **Постусловия** | Подписка активна; премиум-функции доступны |

## 3. Сценарии для региональных партнёров

### UC-09: Создание региональной промоакции партнёром

| Параметр | Значение |
|:---|:---|
| **ID** | UC-09 |
| **Название** | Создание региональной промоакции партнёром |
| **Описание** | Региональный партнёр (фитнес-клуб, спортивный магазин) создаёт промоакцию для пользователей в своём регионе через админ-панель. |
| **Первичный актор** | Партнёр |
| **Вторичные акторы** | Сервис B2B/Partners, Сервис Notification, Админ-панель |
| **Предусловия** | Партнёр зарегистрирован и аутентифицирован в системе (роль "Партнёр") |
| **Основной поток** | 1. Партнёр входит в админ-панель (веб-интерфейс) (ФТ-10).<br>2. Система отображает дашборд с аналитикой.<br>3. Партнёр нажимает "Создать промоакцию".<br>4. Система отображает форму: название, описание, условия (скидка), период действия, регион (геолокация).<br>5. Партнёр заполняет форму и выбирает целевую аудиторию (вид спорта, уровень подготовки, возраст).<br>6. Партнёр нажимает "Опубликовать".<br>7. Система проверяет акцию на соответствие правилам и публикует (ФТ-9).<br>8. Система отображает акцию в ленте у пользователей выбранного региона.<br>9. Партнёр видит статистику: просмотры, переходы, конверсии. |
| **Альтернативные потоки** | 1. Акция отклонена модератором — система отображает причину.<br>2. Партнёр может отредактировать акцию до публикации.<br>3. Партнёр может завершить акцию досрочно. |
| **Постусловия** | Акция создана и опубликована |
| **Бизнес-цель** | БТ-4 (B2B-платформа) |
| **Связанные ФТ** | ФТ-9 (Региональные промоакции), ФТ-10 (Админ-панель) |
| **Связанные НФТ** | НФТ-01 (время отклика), НФТ-53 (документация API) |

### UC-10: Интеграция партнёра через публичное API

| Параметр | Значение |
|:---|:---|
| **ID** | UC-10 |
| **Название** | Интеграция партнёра через публичное API |
| **Описание** | Партнёр (фитнес-клуб, организатор событий) интегрирует свою систему с платформой через публичное API для автоматической синхронизации данных (регистрация на события, импорт данных). |
| **Первичный актор** | Партнёр (разработчик) |
| **Вторичные акторы** | Сервис B2B/Partners, Публичное API Gateway |
| **Предусловия** | Партнёр зарегистрирован в системе; получен API-ключ; документация доступна |
| **Основной поток** | 1. Партнёр-разработчик читает документацию публичного API (OpenAPI/Swagger).<br>2. Партнёр отправляет запрос на регистрацию приложения (получение client_id и client_secret).<br>3. Система генерирует credentials и отправляет партнёру.<br>4. Партнёр разрабатывает интеграцию на своей стороне.<br>5. Партнёр отправляет API-запрос (например, создание события в календаре) с валидным API-ключом.<br>6. Система аутентифицирует запрос и выполняет операцию (ФТ-12).<br>7. Система возвращает ответ в формате JSON.<br>8. Партнёр интегрирует данные в свою систему. |
| **Альтернативные потоки** | 1. API-ключ невалидный — система возвращает ошибку 401.<br>2. Превышен лимит запросов (rate limiting) — система возвращает 429. |
| **Постусловия** | Интеграция настроена; данные синхронизируются |
| **Бизнес-цель** | БТ-4 (B2B-платформа) |
| **Связанные ФТ** | ФТ-12 (Публичное API) |
| **Связанные НФТ** | НФТ-15 (rate limiting), НФТ-53 (документация API) |
